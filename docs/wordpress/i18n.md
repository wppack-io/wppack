# WordPress i18n（国際化）API 仕様

## 1. 概要

WordPress の i18n（国際化）/ l10n（ローカライゼーション）システムは、GNU gettext をベースとした翻訳フレームワークです。`.mo`（Machine Object）/ `.po`（Portable Object）ファイル形式で翻訳データを管理し、テキストドメインごとに翻訳を分離します。WordPress 6.5 からは PHP ファイルベースの翻訳形式もサポートしています。

### 主要クラス

| クラス | 説明 |
|---|---|
| `WP_Translations` | WordPress 6.5+ で導入された翻訳クラス。PHP ファイルベースの翻訳をサポート |
| `WP_Translation_File` | 翻訳ファイルの抽象基底クラス |
| `WP_Translation_File_MO` | `.mo` ファイルパーサー |
| `WP_Translation_File_PHP` | `.php` 翻訳ファイルパーサー |
| `WP_Textdomain_Registry` | テキストドメインとファイルパスのレジストリ |

### レガシークラス（後方互換）

| クラス | 説明 |
|---|---|
| `MO` | レガシー `.mo` ファイルパーサー（`Translations` を拡張） |
| `PO` | `.po` ファイルパーサー |
| `POMO_Reader` | ファイル読み取りの基底クラス |
| `Translations` | 翻訳エントリのコレクション |
| `Translation_Entry` | 個別の翻訳エントリ |
| `NOOP_Translations` | 翻訳を行わないダミー実装 |

### グローバル変数

| グローバル変数 | 型 | 説明 |
|---|---|---|
| `$l10n` | `array<string, Translations>` | テキストドメインをキーとした翻訳オブジェクトの配列（レガシー） |
| `$l10n_unloaded` | `array<string, true>` | アンロードされたテキストドメインのリスト |
| `$wp_textdomain_registry` | `WP_Textdomain_Registry` | テキストドメインレジストリ |

## 2. データ構造

### テキストドメイン

テキストドメインは翻訳をグループ化する識別子です。WordPress コアは `'default'` ドメインを使用し、プラグイン/テーマは独自のドメインを持ちます。

```php
// WordPress コア
__('Hello', 'default');  // 'default' は省略可能

// プラグイン
__('Hello', 'my-plugin');

// テーマ
__('Hello', 'my-theme');
```

### 翻訳ファイルの構造

#### .mo ファイル（バイナリ形式）

```
ヘッダー:
  magic number  (4 bytes) - 0x950412de (LE) or 0xde120495 (BE)
  revision      (4 bytes) - ファイル形式のリビジョン
  num_strings   (4 bytes) - 文字列の数
  originals_off (4 bytes) - オリジナル文字列テーブルのオフセット
  translations_off (4 bytes) - 翻訳文字列テーブルのオフセット
  hash_size     (4 bytes) - ハッシュテーブルのサイズ
  hash_off      (4 bytes) - ハッシュテーブルのオフセット

文字列テーブル:
  各エントリ: length (4 bytes) + offset (4 bytes)

文字列データ:
  NULL 終端の文字列群
```

#### .php 翻訳ファイル（WordPress 6.5+）

```php
<?php
return [
    'translation-revision-date' => '2024-01-01 00:00+0000',
    'generator'   => 'WP-CLI/2.10.0',
    'source'      => 'my-plugin-ja.po',
    'locale'      => 'ja',
    'domain'      => 'my-plugin',
    'messages'    => [
        'Hello'       => 'こんにちは',
        'Goodbye'     => 'さようなら',
        // コンテキスト付き: "\x04" でコンテキストとメッセージを区切る
        "menu\x04File" => 'ファイル',
        // 複数形: "\x00" で単数形と複数形を区切る
        "%d item\x00%d items" => "%d 件",
    ],
];
```

### 翻訳ファイルの読み込み優先順

WordPress は以下の順序で翻訳ファイルを検索します:

1. `WP_LANG_DIR/plugins/{textdomain}-{locale}.mo` (カスタム翻訳)
2. プラグイン/テーマの `languages/` ディレクトリ
3. `WP_LANG_DIR/{locale}.mo` (コア翻訳)

`.php` ファイルが `.mo` ファイルと同じ場所に存在する場合、`.php` ファイルが優先されます（パフォーマンス上の理由）。

### ロケール文字列の構造

WordPress のロケール文字列は以下の形式です:

```
言語[_地域[_バリアント]]

例:
  ja        - 日本語
  en_US     - アメリカ英語
  pt_BR     - ブラジルポルトガル語
  zh_CN     - 簡体中国語
  zh_TW     - 繁体中国語
```

## 3. API リファレンス

### 基本翻訳関数

| 関数 | シグネチャ | 説明 |
|---|---|---|
| `__()` | `(string $text, string $domain = 'default'): string` | 翻訳された文字列を返す |
| `_e()` | `(string $text, string $domain = 'default'): void` | 翻訳された文字列をエコー出力 |
| `_x()` | `(string $text, string $context, string $domain = 'default'): string` | コンテキスト付きで翻訳された文字列を返す |
| `_ex()` | `(string $text, string $context, string $domain = 'default'): void` | コンテキスト付きで翻訳された文字列をエコー出力 |
| `_n()` | `(string $single, string $plural, int $number, string $domain = 'default'): string` | 複数形を考慮した翻訳を返す |
| `_nx()` | `(string $single, string $plural, int $number, string $context, string $domain = 'default'): string` | コンテキスト付き複数形翻訳を返す |

```php
// 基本翻訳
$text = __('Hello World', 'my-plugin');
_e('Hello World', 'my-plugin');

// コンテキスト付き（同じ原文でも文脈で訳し分け）
$text = _x('Post', 'noun', 'my-plugin');    // 名詞としての「投稿」
$text = _x('Post', 'verb', 'my-plugin');    // 動詞としての「投稿する」

// 複数形
$text = _n('%d item', '%d items', $count, 'my-plugin');
$text = sprintf($text, $count);

// コンテキスト付き複数形
$text = _nx('%d post', '%d posts', $count, 'blog posts', 'my-plugin');
```

### テキストドメイン管理

| 関数 | シグネチャ | 説明 |
|---|---|---|
| `load_textdomain()` | `(string $domain, string $mofile, ?string $locale = null): bool` | テキストドメインの翻訳ファイルを読み込む |
| `unload_textdomain()` | `(string $domain, bool $reloadable = false): bool` | テキストドメインをアンロード |
| `load_plugin_textdomain()` | `(string $domain, string\|false $deprecated = false, string\|false $plugin_rel_path = false): bool` | プラグインの翻訳を読み込む |
| `load_theme_textdomain()` | `(string $domain, string\|false $path = false): bool` | テーマの翻訳を読み込む |
| `load_child_theme_textdomain()` | `(string $domain, string\|false $path = false): bool` | 子テーマの翻訳を読み込む |
| `load_default_textdomain()` | `(?string $locale = null): void` | WordPress コアのデフォルト翻訳を読み込む |
| `is_textdomain_loaded()` | `(string $domain): bool` | テキストドメインが読み込み済みか確認 |
| `get_translations_for_domain()` | `(string $domain): Translations\|NOOP_Translations` | テキストドメインの翻訳オブジェクトを取得 |

```php
// プラグインでの翻訳読み込み
add_action('init', function () {
    load_plugin_textdomain('my-plugin', false, 'my-plugin/languages');
});

// テーマでの翻訳読み込み
add_action('after_setup_theme', function () {
    load_theme_textdomain('my-theme', get_template_directory() . '/languages');
});

// 手動読み込み
load_textdomain('my-domain', '/path/to/my-domain-ja.mo');
```

## 4. 実行フロー

### 翻訳関数の処理フロー（`__()` の場合）

```
__($text, $domain)
│
└── translate($text, $domain)
    │
    ├── $translations = get_translations_for_domain($domain)
    │   │
    │   ├── $l10n[$domain] が存在 → そのオブジェクトを返す
    │   │
    │   ├── Just-In-Time 翻訳読み込み（WordPress 4.6+）
    │   │   ├── _load_textdomain_just_in_time($domain)
    │   │   │   ├── $wp_textdomain_registry からパスを取得
    │   │   │   ├── load_textdomain($domain, $mofile)
    │   │   │   └── 読み込み成功なら $l10n[$domain] にセット
    │   │   └── 失敗なら NOOP_Translations を返す
    │   │
    │   └── return $translations
    │
    ├── $translations->translate($text)
    │   └── 翻訳が見つかれば翻訳文字列を返す
    │       見つからなければ原文を返す
    │
    └── apply_filters('gettext', $translation, $text, $domain)
        └── return $translation
```

### テキストドメイン読み込みフロー

```
load_textdomain($domain, $mofile, $locale)
│
├── apply_filters('load_textdomain_mofile', $mofile, $domain)
│   └── ファイルパスを変更可能
│
├── apply_filters('pre_load_textdomain', null, $domain, $mofile, $locale)
│   └── null 以外 → 早期リターン
│
├── .php ファイルの確認
│   ├── $mofile と同パスの .l10n.php を検索
│   └── 存在すれば WP_Translation_File_PHP でロード（高速）
│
├── .mo ファイルの読み込み
│   ├── WP_Translation_File_MO でパース
│   └── WP_Translations にセット
│
├── $l10n[$domain] にセット
│   └── 既存の翻訳がある場合はマージ
│
├── do_action('load_textdomain', $domain, $mofile)
│
└── return true
```

### Just-In-Time 翻訳読み込みフロー（WordPress 4.6+）

```
_load_textdomain_just_in_time($domain)
│
├── $wp_textdomain_registry->has($domain)
│   └── 登録されていなければ false を返す
│
├── $locale = determine_locale()
│
├── $path = $wp_textdomain_registry->get($domain, $locale)
│   └── テキストドメインの翻訳ファイルパスを取得
│
├── load_textdomain($domain, $path . '/' . $domain . '-' . $locale . '.mo')
│
└── return success/failure
```

### ロケール切り替えフロー

```
switch_to_locale($locale)
│
├── 現在のロケールをスタックに保存
│   └── $wp_locale_switcher->switch_to_locale($locale)
│
├── ロケール変更
│   └── $GLOBALS['locale'] = $locale
│
├── 翻訳の再読み込み
│   ├── load_default_textdomain($locale)
│   └── 読み込み済みテキストドメインの翻訳を再読み込み
│
├── do_action('switch_locale', $locale)
│
└── return true

restore_previous_locale()
│
├── スタックから前のロケールを復元
│   └── $wp_locale_switcher->restore_previous_locale()
│
├── 翻訳の再読み込み
│
├── do_action('restore_previous_locale', $locale, $previous_locale)
│
└── return $locale
```

## 5. 複数形の処理

### 複数形ルール

WordPress は言語ごとに異なる複数形ルールをサポートしています。ルールは翻訳ファイルのヘッダー `Plural-Forms` で定義されます:

```
# 日本語（複数形なし）
Plural-Forms: nplurals=1; plural=0;

# 英語（単数形 + 複数形）
Plural-Forms: nplurals=2; plural=(n != 1);

# フランス語（0 と 1 が単数、2 以上が複数）
Plural-Forms: nplurals=2; plural=(n > 1);

# ロシア語（3 種類の複数形）
Plural-Forms: nplurals=3; plural=(n%10==1 && n%100!=11 ? 0 : n%10>=2 && n%10<=4 && (n%100<10 || n%100>=20) ? 1 : 2);

# アラビア語（6 種類の複数形）
Plural-Forms: nplurals=6; plural=(n==0 ? 0 : n==1 ? 1 : n==2 ? 2 : n%100>=3 && n%100<=10 ? 3 : n%100>=11 ? 4 : 5);
```

### _n() の動作

```php
// 内部的な処理
function _n($single, $plural, $number, $domain) {
    $translations = get_translations_for_domain($domain);
    $translation = $translations->translate_plural($single, $plural, $number);
    // Plural-Forms ルールに基づき適切な翻訳形を選択
    return apply_filters('ngettext', $translation, $single, $plural, $number, $domain);
}
```

## 6. WP_Textdomain_Registry

WordPress 6.1 で導入されたテキストドメインレジストリは、テキストドメインのファイルパスを一元管理します:

```php
class WP_Textdomain_Registry {
    protected $current = [];   // ドメイン → ロケール → パスのマッピング
    protected $all     = [];   // ドメイン → パスリスト

    public function has(string $domain): bool;
    public function get(string $domain, string $locale): string|false;
    public function set(string $domain, string $locale, ?string $path): void;
    public function set_custom_path(string $domain, string $path): void;
    public function get_language_files_from_path(string $path): array;
}
```

プラグインやテーマのヘッダーで宣言されたテキストドメインと `Domain Path` が自動的にレジストリに登録されます。
